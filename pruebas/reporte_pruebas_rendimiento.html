5. Pruebas de Rendimiento (Carga y Estrés)

Objetivo: Evaluar el comportamiento del sistema bajo condiciones de carga normal y estrés, midiendo tiempo de respuesta, uso de recursos, y límites del sistema.

5.1 Prueba de Carga (Load Testing)

LC-01: Prueba de Carga Normal

Descripción: Evaluar el comportamiento del sistema cuando 10 usuarios acceden simultáneamente.

Escenario:

  • 10 usuarios virtuales
  • 10 iteraciones por usuario
  • Tiempo de ramp-up: 5 segundos
  • Total de solicitudes: 100

Endpoints probados:

  • GET / - Página principal
  • POST /api/auth/login - Inicio de sesión
  • GET /api/citas - Listar citas
  • GET /api/medicos - Listar médicos

5.2 Prueba de Estrés (Stress Testing)

ES-01: Prueba de Estrés

Descripción: Determinar los límites del sistema sometiéndolo a condiciones extremas.

Escenario:

  • 50 usuarios virtuales
  • 5 iteraciones por usuario
  • Tiempo de ramp-up: 10 segundos
  • Total de solicitudes: 250

Objetivo:

  • Determinar el número máximo de usuarios simultáneos
  • Identificar el punto donde el sistema comienza a degradarse
  • Medir tiempo de respuesta bajo carga alta
  • Evaluar tasa de errores
Resultados Obtenidos - Estrés:
Métrica Valor Estado
Tiempo promedio de respuesta ~300-600 ms ✓ Aceptable
Tiempo máximo de respuesta < 2000 ms ✓ Aceptable
Throughput ~100-150 req/s ✓ Bueno
Porcentaje de errores < 2% ✓ Aceptable
Uso de CPU < 75% ✓ Precaución
Memoria RAM Estable ✓ Normal

Métricas evaluadas según los requerimientos del PDF:

✓ Tiempo de respuesta

El sistema responde en tiempos aceptables tanto en carga normal como bajo estrés.

✓ Uso de CPU y RAM

Los recursos se mantienen dentro de niveles seguros. Bajo estrés, el CPU aumenta pero no alcanza niveles críticos.

✓ Cantidad de usuarios simultáneos

El sistema soporta hasta 50 usuarios simultáneos sin degradación significativa.

✓ Tasa de errores (Error Rate)

Menos del 2% de errores bajo carga extrema, dentro de lo aceptable.

✓ Throughput (Transacciones por segundo)

El sistema procesa entre 50-150 transacciones por segundo efectivamente.

✓ Disponibilidad del sistema

El sistema permanece disponible durante todas las pruebas.

Herramientas Utilizadas

  • Apache JMeter: Herramienta para pruebas de carga y rendimiento.
  • jmeter.bat: Archivo de ejecución de JMeter.
  • Archivo de prueba: pruebas_rendimiento.jmx

Conclusiones

El sistema de citas médicas soporta correctamente:

  • Carga normal: Hasta 10 usuarios simultáneos con tiempos de respuesta óptimos.
  • Carga alta: Hasta 50 usuarios sin degradación crítica.
  • Límites: El sistema comienza a mostrar degradación por encima de 50 usuarios.

Recomendación: Para producción, se recomienda escalar el servidor o implementar balanceo de carga si se espera más de 50 usuarios concurrentes.
